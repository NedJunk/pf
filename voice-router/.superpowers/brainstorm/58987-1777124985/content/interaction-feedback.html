Core Interaction Model

Defining how the tool "thinks" and "responds" during a voice session.

How should the tool provide feedback during a session?

If you're using this to refine requirements or dump ideas, how do you want to know it's following you?

A

Reactive & Invisible

I talk, it listens silently. It only speaks when I ask it a direct question or when I'm done. I don't want it to interrupt my train of thought.

B

Active Facilitator

Like we're doing now: It asks clarifying questions, suggests categories for my brain-dumped ideas, and prompts me to "dig deeper" on vague points.

C

Collaborative Scribe

It provides short verbal confirmations ("Got it," "Added to tasks") while I talk, acting as a background presence that I know is working.
